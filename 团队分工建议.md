## 团队分工建议

| 成员      | 角色         | 主要负责                                |
| ------- | ---------- | ----------------------------------- |
| **成员A** | 后端开发 + 云部署 | 服务器架构、数据库设计、API接口、高并发处理、云端部署与公网IP配置 |
| **成员B** | 算法/模型      | 人脸检测模型选型与部署、人脸编码与检索、照片库管理逻辑         |
| **成员C** | 移动端开发      | 手机端UI、拍照上传、注册/登录/签到页面、前后端联调         |

---

## 各成员本周具体任务

### 成员A（后端+部署）

- 搭建后端框架（推荐 Flask / FastAPI）

- 设计数据库表结构（用户表、签到记录表、人脸特征表）

- 实现注册、登录、注销的API

- 处理图片上传接口，调用成员B的人脸编码服务

- **本周三提交前**：完成云服务器部署，提供公网IP，演示前后端连通

### 成员B（算法）

- 选择人脸检测模型（推荐 MTCNN 或 RetinaFace）

- 选择人脸识别/编码模型（推荐 ArcFace 或 FaceNet）

- 实现人脸检测 → 人脸编码的 pipeline

- 实现照片检索逻辑（待签到照片编码后，与库中人脸特征比对）

- 封装成API供成员A调用（或作为独立服务）

### 成员C（移动端）

- 搭建移动端框架（推荐 Flutter 或 React Native，或微信小程序）

- 实现注册页面（基本信息 + 标准人脸照片上传）

- 实现登录/注销功能

- 实现拍照上传功能

- **本周三提交前**：与成员A联调，演示注册、登录、拍照上传的完整流程
